--- layout: doc title: create-setup description: this task is used to create a project setup. permalink: /create-setup/ category: doc weight: 2 ---

Full name

com.vdoc.maven:vdoc-maven-plugin:{{site.vdoc_maven_plugin_version}}:create-setup

Attributes

  • Requires a Maven project to be executed.
  • Binds by default to the lifecycle phase: package.

Parameters

buildDirectory
Directory containing the generated JAR.
  • java.io.File
  • ${project.build.directory}
dependenciesSetupsGroupIds
dependencies setup should be merged into this project setup
  • java.util.List
  • true
includeDependenciesSetups
dependencies setup should be merged into this project setup
  • boolean
  • true
includeJavadoc
if true test javadoc should be included in setup
  • boolean
  • false
includeOtherModules
other modules should be merged into this project setup.
Warning : if you turn on this option it's highly recommended to used Parallel builds.
Else the module with this option set to true should be the last to be build (to avoid thread locking).
  • boolean
  • false
includeOtherModulesTimeout
how many time this thread should wait for other modules in second.
  • long
  • 30
includeSource
if true source jar should be included in setup
  • boolean
  • false
includeTest
if true test jar should be included in setup
  • boolean
  • true
jarName
Name of the generated JAR.
  • java.lang.String
  • ${project.build.finalName}
libFolder
where found dependency jars
  • java.io.File
  • ${project.build.directory}/lib
packagingType
the project packaging type actually APPS is the only supported value.
  • com.vdoc.maven.plugin.create.setup.enums.PackagingType
  • APPS
project
The Maven project.
  • org.apache.maven.project.MavenProject
  • ${project}
session
The Maven session.
  • org.apache.maven.execution.MavenSession
  • ${session}
setupName
the apps file name without extension. The setup is suffixed with -setup.
  • java.lang.String
vdocHome
the VDoc home folder if set the apps is copied into apps folder.
  • java.io.File
