Full name

com.vdoc.maven:vdoc-maven-plugin:{{site.vdoc_maven_plugin_version}}:generate-classes

Attributes

  • Requires a Maven project to be executed.
  • The goal is thread-safe and supports parallel builds.

Parameters

baseClasses
Base classes must be generated ?
  • boolean
  • true
buildDirectory
Directory containing the generated JAR.
  • java.io.File
  • ${project.build.directory}
customClasses
custom classes must be generated. (it's recommended to turn it on only first time)
  • boolean
  • false
failOnError
can be turn to off to avoid error if the server can't be join
  • boolean
  • true
jarName
Name of the generated JAR.
  • java.lang.String
  • ${project.build.finalName}
login
A VDoc user login
  • java.lang.String
outputDirectory
where classes must be output. You can prefer a specific folder for generated classes but don't forgot to add this folder into compile plugin.
  • java.io.File
  • ${project.build.sourceDirectory}
password
the user's password
  • java.lang.String
project
The Maven project.
  • org.apache.maven.project.MavenProject
  • ${project}
session
The Maven session.
  • org.apache.maven.execution.MavenSession
  • ${session}
vdocURL
the VDoc url where found data model
  • java.lang.String
